### extract_design_spec

Extracts structured design specifications from UI mockup images using Vision LLM analysis.
This tool takes mockup images (generated by `generate_image` or uploaded manually) and produces
4 machine-readable output files that the code agent consumes for TDD-first implementation.

The tool automatically discovers mockup images in the project's `docs/design-mockups/` directory,
or you can specify images explicitly.

**Output files** (written to project root automatically):
1. `design-tokens.json` — W3C DTCG format color/spacing/typography tokens
2. `component-spec.json` — Component anatomy with typed props, variants, slots
3. `section-map.json` — Page section roles (hero, pricing, etc.) with ordering
4. `design-brief.md` — LLM-ready system prompt with build rules for code agent

Args:
- **image_paths**: (optional) Comma-separated list or array of image file paths to extract from. Can be relative to the project or absolute.
- **image_dir**: (optional) Directory containing mockup images. All `.png`, `.jpg`, `.jpeg`, `.webp` files will be discovered. Can be relative to the project or absolute.
- If neither is provided, the tool **auto-discovers** images in these locations (checked in order):
  - `docs/design-mockups/`
  - `docs/mockups/`
  - `docs/brand/`
  - `design-kit/`
  - `mockups/`

> **IMPORTANT**: Always run this tool AFTER `generate_image` to extract structured specs.
> The code agent needs `design-tokens.json` and `component-spec.json` for TDD.

---

#### Workflow

1. **Generate mockups** with `generate_image` (5 pages: design system, homepage, features, pricing, dashboard)
2. **Extract specs** with `extract_design_spec` — auto-discovers images in `docs/design-mockups/`
3. **Save deliverable** with `save_deliverable` (optional — the tool writes files directly)

The extracted specs are the bridge between visual design and code implementation.

---

usage:

~~~json
{
    "thoughts": [
        "Mockups are generated in docs/design-mockups/.",
        "I need to extract structured design data for the code agent.",
        "extract_design_spec will auto-discover the images."
    ],
    "headline": "Extracting design specification from mockups",
    "tool_name": "extract_design_spec",
    "tool_args": {}
}
~~~

**With explicit image directory:**

~~~json
{
    "thoughts": [
        "The mockups were saved to a custom directory.",
        "I'll point extract_design_spec to the right location."
    ],
    "headline": "Extracting design spec from custom mockup directory",
    "tool_name": "extract_design_spec",
    "tool_args": {
        "image_dir": "docs/brand"
    }
}
~~~

**With explicit image paths:**

~~~json
{
    "thoughts": [
        "I want to extract from specific mockup images only."
    ],
    "headline": "Extracting design spec from selected mockups",
    "tool_name": "extract_design_spec",
    "tool_args": {
        "image_paths": ["docs/design-mockups/00-design-system.png", "docs/design-mockups/01-homepage.png"]
    }
}
~~~
