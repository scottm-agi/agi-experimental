## File Architecture Rules

### `layout.tsx` — Server Component ONLY

```
⛔ NEVER add 'use client' to layout.tsx
⛔ NEVER use React hooks (useState, useEffect, useContext) in layout.tsx
⛔ NEVER import client-only libraries in layout.tsx
```

Layout files (`app/layout.tsx`, `app/*/layout.tsx`) are **Server Components**.
They render on the server and wrap all child pages.

If you need interactive behavior in the layout (e.g., a sidebar toggle),
extract it into a Client Component in `components/` and import it:

```tsx
// ✅ CORRECT: layout.tsx imports a Client Component
import { Sidebar } from '@/components/Sidebar'

export default function RootLayout({ children }) {
  return (
    <html><body>
      <Sidebar />  {/* Client Component handles interactivity */}
      {children}
    </body></html>
  )
}
```

### `page.tsx` — Dynamic Rendering for Hooks

If a page uses React hooks, Context, or browser APIs, add:

```tsx
export const dynamic = 'force-dynamic'
```

This tells Next.js to skip static prerendering for this page, which prevents
`Cannot read properties of null (reading 'useContext')` errors.

```tsx
// ✅ CORRECT: page with hooks uses force-dynamic
'use client'
export const dynamic = 'force-dynamic'

import { useState } from 'react'

export default function DashboardPage() {
  const [count, setCount] = useState(0)
  return <div>{count}</div>
}
```

### `components/` — Client Components

All interactive components with `'use client'` belong in `components/` or `app/_components/`.

```tsx
// components/Counter.tsx
'use client'
import { useState } from 'react'

export function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>
}
```

Server-side utilities, database clients, API helpers go in `lib/`:

lib/

### `app/api/` — API Routes

API routes use the Route Handler pattern:

```tsx
// app/api/users/route.ts
import { NextResponse } from 'next/server'

export async function GET() {
  return NextResponse.json({ users: [] })
}

export async function POST(request: Request) {
  const body = await request.json()
  return NextResponse.json({ created: true })
}
```

## Common Error Patterns

| Error | Cause | Fix |
|---|---|---|
| `Cannot read properties of null (reading 'useContext')` | Hook used in Server Component or during static prerendering | Add `export const dynamic = 'force-dynamic'` AND `'use client'` |
| `Module not found: lucide-react` | Package not installed | Run `npm install lucide-react` |
| `Hydration mismatch` | Server/client HTML differs | Ensure dynamic content uses `useEffect` or `suppressHydrationWarning` |
| `Error: Unsupported Server Component type: undefined` | Default export missing | Ensure every page/layout has `export default function` |
